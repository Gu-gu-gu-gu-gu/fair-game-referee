# Fair Game Referee（公平游戏裁定器）

一个用于 [SillyTavern](https://github.com/SillyTavern/SillyTavern) 的第三方扩展，用于在多人互动游戏（飞行棋、骰子、国王游戏、真心话大冒险、俄罗斯转盘）中提供**可追踪、可约束、可复现**的回合裁定结果，并将结果注入上下文，减少模型“改判/跳轮/偷跑新一轮”的问题。

---

## 功能简介

- 回合触发检测（本地关键词 + 可选外部 LLM 判定）
- 游戏类型识别与切换
- 飞行棋回合运算（顺序、移动、事件、碰撞、终点）
- 骰子回合运算（支持自动/固定骰子数量）
- 裁定结果注入（严格模式 / 导演模式 / 仅展示）
- 地图可视化编辑（格子事件、导入/导出/重命名/删除）
- 角色维度玩家名单（支持写入角色扩展字段）
- 长按扩展按钮快速打开面板（可开关）
- 回退上一轮 / 前进下一轮（撤销/重做）

---

## 安装方式

- 打开 SillyTavern 的 Extensions 界面，选择“Install extension”
- 粘贴本仓库的 URL，点击“Install”

---

## 快速开始

1. 在扩展设置里启用 **公平游戏裁定器**
2. 打开“公平裁定面板”
3. 在“编辑”页设置玩家名单（建议至少 2 人）
4. 在“地图”页确认飞行棋地图（终点、事件）
5. 聊天中发送触发词（如“飞行棋”“下一轮”）进行裁定

---

## 回合与规则说明（核心）

- **一轮**：所有玩家各执行一次掷骰/行动，并结算各自落点任务
- **下一轮**：在上一整轮结束后，全员再进行一次新的掷骰/行动
- **不是重开**：除非用户明确说“重玩/重开/重新开始”，否则不应回到起点重头
- 飞行棋碰撞遵循先后顺序：先行动者先完成其任务；后续玩家若撞到其当前所在格，会把其送回起点并结算起点事件（不撤销其此前已完成内容）

---

## 外部判定（可选）

支持：

- OpenAI 兼容接口
- Google AI Studio

可在设置页配置：

- Provider
- Endpoint / API Key / Model
- 是否每条用户消息都调用判定 API

> 注意：部分接口可能受 CORS 或 models 列表权限限制。

---

## 地图系统

- 可视化编辑格子事件（`at`, `move`, `text`）
- 支持单地图或地图包导入导出（JSON）
- 支持地图重命名、删除
- 支持保存为新地图（避免覆盖）

---

## 已知注意事项

- 玩家少于 2 人不会启动回合裁定
- 若使用外部判定，网络异常会回退到本地关键词判定
- 不同模型对“任务叙事”执行度不同，建议使用严格注入约束
