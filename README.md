# 游戏裁定器（SillyTavern 扩展）

> 面向多人桌游 / 飞行棋 / 骰子剧情的回合裁定辅助扩展。
> 目前主要支持 **飞行棋**，内置 d6 / 2d6 自动切换逻辑，并提供可视化棋盘与“导演模式”回合微调。

---

## 功能特性

- 🎲 **飞行棋裁定**
  - 自动识别玩家名单（支持手动名单 + 聊天记录中的说话人）
  - 支持起点/终点、自定义格子事件（任务 / 道具等）
  - 单次掷骰会自动：
    - 为所有玩家掷骰
    - 结算位移、相撞回起点、起点事件、终点胜利等

- 🎲 **d6 / 2d6 自动切换**
  - 骰子数量模式：
    - `auto`：根据玩家数自动切 1d6 / 2d6
    - `fixed`：强制 1d6 或 2d6
  - 所有随机在本地完成，不依赖外部 API，**不消耗额度**

- 🧠 **回合触发逻辑**
  - 通过关键词自动识别：
    - “飞行棋 / 玩飞行棋 / 开始玩飞行棋”等 → 开启/切回飞行棋
    - “下一回合 / 新一回合 / next round”等 → 进入下一回合（保持当前游戏）
    - “重玩 / 再玩一次 / 重新开始”等 → 重置飞行棋进度重新开局
  - 你可以在设置里自定义这些触发词

- 🗺️ **可视化地图编辑**
  - 内置简易棋盘编辑器：
    - 设置终点格（winPosition）
    - 为指定格子定义 move（位移）+ text（事件说明）
  - 支持：
    - 导出单张地图 / 整个地图包（JSON）
    - 导入地图 / 地图包（自定义任务集）

- 🎬 **导演模式（回合微调）**
  - 每次裁定后，可以打开“导演编辑”小面板：
    - 查看本回合所有玩家落点
    - 手动微调玩家落点（比如纠正角色扮演中的特别需求）
    - 一键“应用到回合包”，自动更新系统提示词
  - LLM 只会按照 **最终裁定结果** 叙事，而不会改判胜负

- 🧩 **SillyTavern 原生集成**
  - 在扩展侧边栏可以启用/禁用插件
  - 长按扩展按钮可快速打开飞行棋面板（可在设置中关闭）

- ⚡ **消息侧快捷操作（新增）**
  - 在**最后一条 AI 消息**旁显示 3 个内联按钮：
    - `✈` 开始飞行棋
    - `♚` 开始国王游戏
    - `⏭` 下一回合
  - 按钮点击走插件内部逻辑，不发送用户文本，避免“下一回合”被模型当作用户台词写入剧情

## 安装方式

### 方式一：SillyTavern 内置安装（推荐）

1. 打开 SillyTavern 前端页面
2. 进入菜单：`Extensions`
3. 点击右上角 `Install extension`
4. 在弹出的输入框中粘贴本仓库的克隆 URL。
5. 点击 `Install`
6. 安装完成后，**刷新浏览器页面**，在 `Extensions` → `Third-party` 里找到并启用 **“游戏裁定器”**

> 如果使用内置安装，SillyTavern 会自动把仓库克隆到
> `SillyTavern/public/scripts/extensions/third-party/fair-game-referee/`

---

### 方式二：手动安装

1. 在 GitHub 上下载本仓库的 ZIP 压缩包并解压
2. 把整个文件夹放到 SillyTavern 扩展目录下：SillyTavern/public/scripts/extensions/third-party/fair-game-referee
3. 刷新 SillyTavern 页面

---

## 使用说明

### 1. 启用插件

1. 在 SillyTavern 前端点击 `Extensions`
2. 在扩展列表中找到 **“游戏裁定器”**
3. 勾选「启用插件」

插件启用后，会自动接管：

- 飞行棋相关关键词（启动 / 重开）
- “下一回合 / next round”等回合触发词

---

### 2. 打开飞行棋面板

有两种方式：

- **方式 A：侧边栏按钮**
  1. 在 `Extensions` 设置中，点击 **“打开游戏裁定面板”** 按钮
  2. 会弹出一个带有“地图 / 编辑 / 设置”三个 Tab 的主面板

- **方式 B：长按扩展按钮**
  1. 在聊天界面左侧工具条，长按扩展菜单按钮（默认 0.55 秒）
  2. 会直接打开飞行棋地图 Tab
  3. 你可以在设置里关闭/开启“长按打开面板”功能

---

### 3. 开始一局飞行棋

1. 确保插件已启用
2. 在聊天里让用户说出类似：
   - `我们来玩飞行棋`
   - `玩飞行棋`
   - `开始玩飞行棋`
3. 插件会：
   - 自动识别当前聊天中的玩家（包括 user + 角色名 + 手动名单）
   - 初始化飞行棋进度（如果是首次）
   - 裁定第一回合结果，生成「回合包」提示词发送给 LLM

之后：

- 只要用户说出 “下一回合 / 新一回合 / next round”等，插件就会自动裁定下一回合。
- 说出“重玩 / 再玩一次 / 重新开始”时，会重置棋子进度，从起点重新开局。

---

### 4. 点击摇骰子（地图上的杯子）

在飞行棋地图 Tab 中：

1. 中间有一个透明玻璃骰盅，里面是 1~2 个骰子
2. **点击骰盅**，会触发一次飞行棋回合裁定：
   - 对所有玩家掷骰（遵从 d6 / 2d6 模式）
   - 通过动画播放掷骰和棋子移动过程
   - 更新当前聊天的飞行棋状态与裁定提示词

如果上一回合已经有人到达终点，会弹出一个简单选择：

- 输入 `1`：在当前盘的基础上继续“下一回合”（只对还没结束的玩家继续跑）
- 输入 `2`：重置棋盘，重新开一盘

---

### 5. 导演模式

在设置页把结果模式设为 **“导演模式”** 时：

- 每次完成回合裁定后，插件会自动弹出一个**小的“导演编辑”独立面板**：
  - 只包含本回合玩家列表、落点输入框和事件文本预览
  - 你可以调每个玩家的最终格子位置
  - 点击「应用到回合包」后：
    - 会重新生成该回合的系统提示词
    - 不会改写历史聊天，只影响当前及后续 LLM 叙事

### 6. 快捷按钮

聊天区会在**最后一条 AI 消息**右侧操作区附近显示三个按钮：

- `✈`：开始飞行棋（内部开局）
- `♚`：开始国王游戏（内部开局）
- `⏭`：推进下一回合（内部结算并更新回合包提示词）

说明：

- 这三个按钮都不会向聊天发送“用户文本消息”
- 因此不会污染剧情，不会让 AI 把“下一回合”当作用户发言来续写

## 设置说明

### 1. 玩家名单（编辑 Tab）

- 支持按角色名自动切换不同的玩家名单配置
- 可以：
  - 新建名单
  - 重命名
  - 删除
  - 切换到“当前角色”的专用名单
- 每个名单里填入“玩家名列表”（换行或逗号分隔）

插件在识别玩家时，会综合：

- 当前名单中的玩家名
- 当前聊天里出现过的消息发送者名
- User 的别名（如：你 / 我 / user / 用户 等）
- 角色名（可选）

---

### 2. 触发词（设置 Tab）

- **下一回合触发词**
  默认：`下一回合,新一回合,next round`

- **飞行棋启动词**
  默认：`飞行棋,玩飞行棋,开始玩飞行棋`

- **飞行棋重开词**
  默认：`重玩,再玩一次,重新开始`

你可以按自己的习惯改成别的，例如：

- `下一轮,继续骰子,继续游戏` 等等

---

### 3. 骰子数量模式

在设置 Tab 中：

- **骰子数量模式**
  - `自动（按玩家数切换）`：当玩家数 ≥ 阈值时使用 2d6，否则 1d6
  - `固定（手动指定）`：强制使用 1d6 或 2d6

- **固定模式骰子颗数**：1 ~ 2
- **自动模式切换阈值**：玩家数达到该值时，用 2d6（默认 6 人）

> 当前版本中，“骰子”游戏本身尚未单独启用，骰子只是作为飞行棋内部的掷骰逻辑存在，**不会单独触发“谁谁谁摇骰子比大小”的剧情**，避免乱入。

---

### 4. 地图编辑与导入导出

在地图 Tab 中你可以：

- 修改终点格 `winPosition`
- 为指定格子添加事件：
  - `move`：整数位移（+前进，-后退）
  - `text`：事件文字说明（任务、道具、惩罚等）
- 用右上方的小图标：
  - 导入地图 JSON / 地图包 JSON
  - 导出当前地图
  - 导出全部地图为一个地图包
  - 重命名 / 删除地图

导出格式是标准 JSON，可以方便你在不同 SillyTavern 实例之间迁移自己的任务地图。
